# Audio

播放音频。

# 方法

# load(options, callback)

加载音频。前期准备工作，例如下载播放资源，但是不会播放。

#参数

options.id {String} 当前playback的id，因为支持同时播放多个，所以id必须唯一，否则会复用原id的播放资源；
options.url {String} url；
options.autoplay {Boolean} load完成后是否自动开始播放。从而不需要触发play()方法，默认值：false；
options.loop {Boolean} 循环播放，默认值：false；
options.volume {String} 声音大小，默认值：1。
callback {Function} 用户操作完成后的回调，回调函数的参数 ret 的格式如下：
ret.id {String} 当前playback的id，原样返回给前端；
ret.status {String} 当前播放器状态：
1：INIT 初始化
2：READY 加载完成，options里面url对应的音频文件已经下载完成
3：PLAYING 播放中
4：PAUSE 暂停
5：ENDED 播放结束，loop为true的，单次播放结束不会触发ENDED的发回调
6：ERROR 错误
ret.value {Object or String} 当前状态的详细信息。
注意：status=2时可以通过value.duration获取音频时长；status=6时则返回一个object，例如{code: 2, message: 'error message'}
error code说明：
1：ABORTED 用户取消
2：NETWORK 网络相关
3：DECODE 解码错误
4：SRC_NOT_SUPPORTED 文件格式不支持
5：OTHER 其他错误

# 示例

const audio = weex.requireModule('audio') 
const option = {
	"url": "http://flv2.bn.netease.com/videolib3/1611/01/XGqSL5981/SD/XGqSL5981-mobile.mp4",
	"id": 1001
}
audio.load(option, function(param) {
	const playerId = param.id;
	console.log(JSON.stringify(param));
})

# play(id)

播放音频。

# 参数

id {String} load()方法的callback中返回id。

# 示例

const audio = weex.requireModule('audio') 
audio.play(1)

# pause(id)

暂停播放。

# 参数

id {String} load()方法的callback中返回id。

# 示例

const audio = weex.requireModule('audio')
audio.pause(1)

# stop(id)

暂停播放。

# 参数

id {String} load()方法的callback中返回id。

# 示例

const audio = weex.requireModule('audio') 
audio.stop(1)

# setVolume(volume)

设置音量。
注意：这里仅仅是app自身的音量大小，和系统音量没关系。

# 参数

volume {Number} 0-1的数值，0=没声音，1=最大。

const audio = weex.requireModule('audio') 
audio.setVolume(0.8)

# canPlayType(mediaType)

当前终端能否播放该音频类型。

# 参数

mediaType {String} 可选值：audio/wav audio/x-wav audio/amr audio/mp3。

# 返回值

{String} 一定可以播放此类型的音频，可选值：probably（一定可以播放此类型的音频），maybe（可能可以播放此类型的音频）''（一定不能播放，例如 mediaType 为“application/octet-stream”）。

const audio = weex.requireModule('audio') 
const result = audio.canPlayType('audio/mp3')
