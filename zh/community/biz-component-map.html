# 简介

经过多年业务沉淀，我们带来了数据驱动的配置化图表。通过提供配置对象展示纯Native图表，性能比市面的图表都要好。支持类型也较为丰富，Native图表沉淀的图表库为AceChart。所以WeexAceChart顾名思义是由AceChart做底层支撑。中间经过了WeexPlugin层，注册Component到weex环境中。因此接入WeexAceChart图表较为简单：

  1. 两端Native接入对应WeexPlugin。
  2. js端接入使用Weex图表的基于Vue框架的库。

目前支持的图表类型如下：

  • 线图
  • 柱图
  • 散点图
  • 气泡图
  • 饼图
  • 玫瑰图
  • 雷达图

并且，处理了一些目前市面上Native图表所没处理的一些问题，如“Dataset overshooting”问题等等。

商业图表组件是EMAS产品配套的一个服务，提供客户端专业的图表组件，使用商业图表组件需要单独购买。

可扫码安装以下二维码，安装EMAS演示App，体验商业通用组件的效果。

# SDK 集成

# iOS图表集成

集成iOS图表比较简单，只需要在podfile中添加下面的依赖就好

pod 'WeexAceChart'

即可完成集成。

安装会添加以下几个库：

  • WeexAceChart
  • AceChart
  • WeexPluginLoader

如果应用禁用了自动注册组件。可以用以下的方法来手动初始化图表。

#import <WeexAceChart/WXAceChartRegiste.h>
// 在WeexSDK初始化后调用这个函数
WeexAceChartRegiste();

注册结束后就完成注册了。

# Android图表集成

# 1. 在build.gradle 的dependencies中添加一下依赖

maven指定为http://nexus-ce.emas-poc.com

compile 'org.weex.plugin.weexacechart:wx_acechart:0.0.2@aar'
compile 'com.alibaba.dt.opm:acechart:1.1.2.28@aar'

# 2. 若发现有类找不到的情况（因为wx_acechart包含有provided依赖），可以根据应用使用库版本在外部引入

//包含有weex_sdk、annotation和weex plugin processor依赖
provided "com.android.support:support-annotations:${your_project_version}"
provided "com.taobao.android:weex_sdk:${your_project_version}"
provided 'org.weex.plugin:processor:1.0.2'

# 3. 组件采用plugin的自动注册组件，加入依赖，并在weex工程中引入AceChart JS库，即可开始使用acechart表，如果plugin注册失败请手动进行组件注册

WeexPluginContainer.loadAll(getApplicationContext());//这段执行失败后使用下面代码
/**---------------------------------
    WXSDKEngine.registerComponent("acechartandroid", WXAChartComponent.class);
    WXSDKEngine.registerComponent("acechartandroidrose", WXAceRoseChartComponent.class);
    WXSDKEngine.registerComponent("acechartandroidpie", WXAcePieChartComponent.class);
    WXSDKEngine.registerComponent("acechartandroidstack", WXScatterPlotChartComponent.class);
*/

# Weex集成WeexAceChart

仅仅在iOS和Android客户端中进行SDK的集成还不够完整。因为两端实现存在一些差异，需要一个用于统一差异的js。因此需要再在 package.json 中，再引入一个库 （https://www.npmjs.com/package/weex-acechart-js）：

weex-acechart-js

执行以下脚本

npm install --save weex-acechart-js@latest
